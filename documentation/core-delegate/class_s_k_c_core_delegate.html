Detailed Description

DESCRIPTION

This object provides functions related vtView features. Key functions are listed below: Identify Device: Retrieve information about the device, supported features, and its state (features are supported and enabled).

A typical sequence is:

  1. Scan for list of devices
  2. Set up logging parameters: frequency of logging, period for compute and send result, select drive(s) to analyze
  3. Start the logging
  4. Stop the logging
  5. Use vtView API to load data and analyze

Method to add a header line to log file.

Parameters
deviceSetA list of strings of devices name.

fileNameA string of log file name (if log file not exis it will create new file, otherwise will append to the end).
Returns
The error enum with one of the following specified error:
  • eErrorNone when success.
  • eSerialNotFound when serial number not found.
  • eRecordNotFound when log record not found.
  • eAlreadySchedule when there is a logging session already running.
  • eNoLogFileSpecified when no log file specified.
  • eDriveUnplugged when given driver disconnected or unplugged.
  • eFailToCreateLogFile when failed to create log file.
  • eFailToSchedule when failed to start a logging session.
  • eDeviceNotFoud when given device not found.
Note
Need to call StopLogging() or ReleaseLogData() first if there is a running session, if not it will wait for current session finished.

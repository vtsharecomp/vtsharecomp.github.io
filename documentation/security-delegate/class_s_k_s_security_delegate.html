Detailed Description

DESCRIPTION

This class supports following functions:

  • Secures erase a SSD. Secure Erase resets security and erase all data (user data, marked bad blocks, and spare blocks).
  • Sanitizes a SSD. Sanitize erases all data (user data, marked bad blocks, and spare blocks).
  • PSID-Reverts reverts TCG OPAL setting to manufacturing default and erases data.
  • OPAL setup device (PSID-revert only be able to performed if OPAL setup device initialized).
  • Security supports password management, manage master password, and user password (set password, disable password, and unlock disk).

virtual SKSSecurityReturnCode SKSSecurityDelegate::sanitizeBlockErase ( )

Sanitizes a SSD with Block Erase command.

This operation uses the block erase method on the user data ereas, including user data areas that are not currently allocated to cause the user data to become unretrievable. Block Erase will erase all physical NAND blocks.

Returns
The security return code enum with one of the following specified code:
  • SKSSucceeded when success.
  • SKSErrorCommandFailed when command IO send failed.
  • SKSErrorFeatureNotSupported when device is not supported the sanitize block erase feature.
  • SKSErrorSecurityLocked when security is disabled and security is in locked state.
  • SKSErrorDeviceFrozen when device security is in frozen state.
Note
Not all drives support this mode. Check for the supported modes and whether the mode is enabled before issue call to sanitize for this mode.
If returning error SKSErrorDeviceFrozen then please unfrozen the drive by hardware reset then try again.
If returning error SKSErrorSecurityLocked then please use the function securityUnlock to unlock disk then try again.

This operation uses to changes the internal encryption keys that are used for user data causing the user data to become unretrievable. This method will change the cryptographic key on the SSD. (This will cause the data to be scrambled. The physical NAND is not erased.)

Secures Erase a SSD.

Methods to initiate secure erase on a storage device.

There are two (2) erase modes: Normal and Enhanced. These modes are passed into the function as a parameter.

A password is required for a secure erase. There are two (2) types of passwords: User and Master.

Each password type results in a different outcome.

Please reference the ATA Specification for information on which to use and when to use a master or user password.

Parameters
modeA SKSecureEraseMode object to specify the secure erase mode (normal or enhanced).
passwordA string of charaters used to verify the identify of a user.
isMasterPasswordA bool to determine if password type is master, otherwise user.
Returns
The security return code enum with one of the following specified code:
  • SKSSucceeded when success.
  • SKSErrorCommandFailed when command IO send failed.
  • SKSErrorFeatureNotSupported when device is not supported the secure erase feature.
  • SKSErrorPasswordUnidentifier when security is disabled and password type is user.
  • SKSErrorDeviceFrozen when device security is in frozen state.
  • SKSErrorNotAuthorized when password is not authorized.
Note
If returning error SKSErrorDeviceFrozen then please unfrozen the drive by hardware reset then try again.
